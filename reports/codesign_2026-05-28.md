### Core Signals (Cross-Sectionally Ranked & Z-Scored)
To replace the unavailable intraday/dark-pool data, we construct four EOD-computable factors:

1.  **Accumulation Volume Asymmetry (AVA):** An EOD proxy for institutional flow.

3.  **Squeeze Fuel (Short Interest):**

### Fix 1: AVA formula has a divide-by-zero / asymmetry bug

`Σ$Vol(up days) / Σ$Vol(down days)` blows up when down-day volume is tiny and is unbounded above. It also conflates "no down days" (bullish) with "small down days" (also bullish) into the same infinity.

**Replacement — signed dollar-volume z-score:**
```
SDV_t = sign(return_t) * dollar_volume_t
AVA_21 = mean(SDV over 21d) / stdev($dollar_volume over 21d)
```
Bounded, symmetric, well-defined when all returns same-sign, and ranks cleanly cross-sectionally. This is the EOD-legal cousin of on-balance-volume normalized by activity.

### Fix 2: Amihud "drop" signal needs a co-movement constraint

`-Amihud_5d / Amihud_63d` flags *any* liquidity expansion — including distressed liquidations (price down, volume up = Amihud also drops). We want **absorption**, not capitulation.

**Replacement:**
```
LiqShock = z(-log(Amihud_5d / Amihud_63d)) * I(ret_5d > 0)
```
The indicator zeros out the down-side-volume liquidations. Log-ratio handles the heavy tail. Already gated by `ret_5d > 0` in eligibility, but making it explicit in the score prevents future eligibility loosening from leaking the bug.

### Fix 3: Short interest is **semi-monthly with a 2-week reporting lag**

Cross-sectional rank of SI%float at rebalance date uses data ~14-28d stale. That's actually fine for 21d holds (SI changes slowly), but we must **use the SI snapshot dated ≤ rebalance_date - 14d** in code to avoid lookahead. Flag for implementation.
